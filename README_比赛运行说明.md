# 弈衡五子棋智能博弈程序

## 比赛运行说明

### 一、项目信息

- 项目名称：弈衡五子棋智能博弈程序
- 算法版本：RCTS-AB 2.2
- 支持规则：指定开局、三手交换、五手N打、禁手规则
- 运行方式：双击 `run_gui.bat` 或 `run_competition.bat`

### 二、线上初赛推荐运行方式

**图形比赛界面（推荐）：**
```
双击 run_gui.bat
或运行：python competition_gui.py
```

**终端界面（备用）：**
```
运行：python main.py
```

### 三、图形比赛界面功能

| 功能 | 说明 |
|------|------|
| 双方15分钟计时 | 独立计时，实时显示 |
| 我方执棋/当前行棋方 | 清晰标识 |
| 指定开局 | 26种开局可选（斜月、名月、寒星等） |
| 五手N打 | N=2/3/4/5可选，程序自动生成候选点 |
| 三手交换 | 程序自动判断是否交换 |
| **禁手检测** | 黑棋三三、四四、长连禁手，图形界面可视化 |
| 棋谱保存 | C5格式，可指定文件名 |

### 四、禁手检测功能

图形界面提供禁手检测可视化功能：

1. **禁手规则说明**：显示黑棋三三禁手、四四禁手、长连禁手
2. **禁手状态显示**：实时显示当前是否有禁手
3. **禁手点可视化**：点击按钮可显示所有黑棋禁手点位置（红色×标记）
4. **自动检测**：每步落子后自动检测是否出现禁手

**注意**：
- 禁手仅适用于黑棋，白棋无禁手
- 三四不作为禁手
- 黑棋刚好五连优先胜利，不算禁手
- 禁手判断由程序自动完成，操作员不进行人工判断

### 五、比赛规则说明

#### 1. 指定开局

根据老师确认，**指定开局名称由执黑方自行选择，五手N打N值在开局阶段由执黑方选定**，N范围为2-5。因此图形界面提供开局名称和N值选择功能。

- 黑1必须为H8（天元）
- 白2必须在H8的8个紧邻点（G7/I7/G8/I8/G9/H9/I9）
- 黑3必须在中心5×5区域（F6-J10）
- 程序内置26种开局名称（斜月、名月、恒星等）
- 该选择属于规则允许的开局设定

#### 2. 三手交换

- 黑3落子后，白4之前可执行三手交换
- 交换后双方执棋颜色互换
- 计时器归属不变（我方时间仍是我方）
- **是否交换由程序自动判断**（执白时）

#### 3. 五手N打

- 黑5为N打点（N值由黑方开局时选定，默认N=2）
- 程序自动生成N个候选点
- 对方选择保留一个点落子
- 其他候选点不落到棋盘上
- **候选点生成和保留点选择由程序自动完成**

#### 4. 禁手规则

- 黑棋三三禁手
- 黑棋四四禁手
- 黑棋长连禁手（>5连）
- 白棋无禁手
- 黑棋刚好五连优先胜利

### 五、操作流程

#### 我方执黑时：

1. 选择"我方执黑"
2. 从下拉框选择开局名称（如：寒星）
3. 从下拉框选择五手N打N值（2/3/4/5）
4. 点击"程序生成指定开局"
5. 程序自动显示前三手
6. 等待对方选择是否交换
7. 程序自动生成五手N打候选点
8. 输入对方选定的保留点
9. 正常对弈直至结束

#### 我方执白时：

1. 选择"我方执白"
2. 输入对方给出的前三手（如：B(H,8);W(I,8);B(H,9)）
3. 程序自动校验并**由程序自动判断是否交换**
4. 如果交换，更新执棋颜色
5. 正常对弈直至结束

### 六、操作员职责

**允许的操作员行为：**
- 选择开局名称和N值（规则允许的开局设定）
- 输入对方程序给出的落子
- 输入对方给出的前三手/候选点
- 点击确认、保存等操作

**不允许的操作员行为：**
- 手动替程序选择我方落子
- 手动判断是否三手交换
- 手动选择五手N打保留点
- 手动判断禁手
- 手动修改棋盘

### 七、计时器说明

- **我方剩余时间**：我方程序思考时递减
- **对方剩余时间**：等待对方输入时递减
- **当前行棋方**：标识当前应落子方
- **我方执棋**：始终显示我方原始颜色
- **暂停/继续**：可暂停计时进行其他操作
- **重置计时**：将时间重置为15:00

### 八、注意事项

1. **不要修改源码**：保持程序稳定性
2. **不要连接网络**：防止外部干扰
3. **准确输入**：对方落子格式如 `B(H,8)` 或 `H8`
4. **及时保存**：对局结束后保存日志和棋谱
